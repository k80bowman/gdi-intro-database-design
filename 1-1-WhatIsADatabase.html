What is a database?

A set of related data and the way it is organized!

What is a database?

A collection of tables, schemas, queries, reports, views, and other objects (it’s not just tables of data!) often organized in a way that supports processes or reflects real life in order to facilitate business or functions.

What is a database - what are those things?!

A collection of tables, schemas, queries, views…

  • Table = collection of related data arranged in columns and rows.
  • Schema = logical grouping of objects (tables, etc.), often in a functional way.
  • Query = request for information from a database.
  • View = result of a stored query.
    • So for 1 database you may have an HR schema and a sales schema. A view looks like a table, but a view’s data is calculated in real-time. Also provide pretty significant advantages to a table - can restrict access to just views (can be a subset of data), can simplify or aggregate data, take very little space to store since a database stores just the definition of the view, not a copy of all the data it has.

Levels of Data

External level = How each group of end-users sees the organization of data in the database. Can be any number. Conceptual/logical level = Unifies all of the external views. This is the level you would use to write code against. Only 1. Internal level = Internal organization of data in a DBMS. Only 1.

Data Independence

Changes made at one level do not affect a view at another level.

So, for example, an end user can change their external view and it doesn’t affect a database application developer’s code at the conceptual level! So you can make physical changes to the database, for optimization for example, without changing the application. If we add some new columns or remove some columns from a table then the user view and programs should not change.

Independence hides the details of the storage structure from user applications.


  • Physical independence = ability to modify the physical schema without causing application programs to be rewritten.
  • Logical independence = ability to modify the logical schema without causing application programs to be rewritten.
