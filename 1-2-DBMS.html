How do I work with a database?

Use a database management system - a DBMS!

  • Connects the data, the user, and other applications.
  • Definition of the database!
  • Creation of the database!
  • Queries of the data!
  • Updates to the data!
  • Administration of the database!

What does a DBMS do for me?

  • Levels of data
  • Storage
  • Data integrity
  • Data consistency
  • Performance
  • Security
  • Backup and recovery
  • And some other stuff...

Levels of Data

Three views provided by the DBMS

  • External level
    • How each group of end-users sees the organization of data in the database.
    • Can be any number of these - allows users to see their own businessy ways!
  • Conceptual/logical level
    • Unifies all of the external views.
    • If you’re a database application developer, this is the view you would use to write code against
    • Only 1
  • Conceptual/logical level
    • Internal organization of data in a DBMS.
    • Only 1

Data Independence

Changes made at one level do not affect a view at another level.

Independence hides the details of the storage structure from user applications.


  • Physical independence = ability to modify the physical schema without causing application programs to be rewritten.
  • Logical independence = ability to modify the logical schema without causing application programs to be rewritten.

Storage

The internal level of the database architecture + metadata and data structures.

Serious optimization happens at the storage/internal level of the database so that the database is fast and efficient when data is requested.


  • Database engine = underlying software a DBMS uses to manipulate the data.
  • Database replication = replicating data in order to increase availability and resiliency.

Data Integrity

Ensuring valid data.

  • Data types ensure that, for example, date fields only store date values.
  • Constraints and triggers allow for complex rules to be applied
    • Constraint = limit the type of data that can go in a table, ensuring accuracy and reliability.
      • Can be column-level or table-level.
      • examples: Not Null, Unique, Primary Key, Default
    • Trigger = action (code) that is run automatically in response to a certain event.
