Jobs for code in databases

Different languages have different functions/roles!

Jobs for code in databases

Data definition language - used to define and modify data structures.

  • CREATE TABLE my_table ( .... )
Examples are in SQL.

Jobs for code in databases

Data manipulation language - used to create and modify data.

  • INSERT INTO my_table VALUES ( .... )
  • UPDATE my_table SET column1 = value1, column2 = value2, ...
  • DELETE FROM my_table WHERE ...

Jobs for code in databases

Query language - used to make queries!

  • SELECT my_column FROM my_table
  • ...WHERE Country='Mexico'
  • ...WHERE Country='Germany' AND City='Berlin'
  • ...ORDER BY Country
  • SELECT AVG(Price)

Jobs for code in databases

Data control language - control access to data/views/etc.

  • REVOKE SELECT ON my_table FROM bad_user

Jobs for code in databases

  • Data definition language
  • Data manipulation language
  • Query language
  • Data control language

Stored procedure

= a block of code created and stored in the database

A stored procedure is like a function that lives in your database. So if you have a specific procedure you have to repeat over and over, or if multiple people need to use, you can keep a stored procedure. Also like functions, stored procedures can have parameters. So my database at work has a stored procedure which takes a loan number as a parameter, and then moves all of the data for that loan into our archive tables.
