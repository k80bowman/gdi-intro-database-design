Where’s the code?

Different languages have different functions/roles:

  • Data definition language - used to define and modify data structures
    • CREATE TABLE my_table ( … )
  • Data manipulation language - used to create and modify data
    • INSERT INTO my_table VALUES ( .... )
  • Query language - used to make queries!
    • SELECT my_column FROM my_table
  • Data control language - control access to data
    • REVOKE SELECT ON my_table FROM bad_user
Examples are in SQL. Different languages will have different roles.

With this code I can...

  • Retrieve data
    • WHERE Country='Mexico'
    • WHERE Country='Germany' AND City='Berlin'
    • ORDER BY Country
    • SELECT AVG(Price)
    • Including joining tables
      • Join = combine rows from two or more tables based on a related column between them
  • Insert records, update, and delete records
  • Create databases
  • Create tables
  • Create views and stored procedures
    • Stored procedure = a block of code created and stored in the database
  • Set permissions on tables, procedures, and views
Verbal example of join. A stored procedure is like a function that lives in your database. So if you have a specific procedure you have to repeat over and over, or if multiple people need to use, you can keep a stored procedure. Also like functions, stored procedures can have parameters. So my database at work has a stored procedure which takes a loan number as a parameter, and then moves all of the data for that loan into our archive tables.
